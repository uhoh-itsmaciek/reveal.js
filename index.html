Augmenting your Schema with JSONB

A case study

Why schemaless?

  • complementary to relational schemas
  • context-specific attributes
  • heterogeneous data
  • experiments
  • schema is the foundation and structural; jsonb the furniture
  • Shogun has 170 different keys in attrs -- don't want 170 different columns
  • e.g., Sequel STI
  • modifying your schema is a mental barrier to trying things

Why JSONB?

  • flexible
  • powerful indexing
  • great performance
  • full json support* (does not support duplicate keys)
  • GIN indexes
  • fast reads and writes

Shogun before

  • hstore (predates jsonb)
  • type inference (sort of)
  • somewhat error-prone

Shogun before


class Foo < Sequel::Model
  def update_foo
    # do some stuff          
    self.attrs[:foo_updated_at] = Time.now
    save_changes
  end
end
  • Sequel does not see the object as modified unless the reference changes
  • only considers the object identity

Shogun before


class Foo < Sequel::Model
  def update_foo
    # do some stuff          
    self.attrs[:foo_updated_at] = Time.now
    modified! :attrs        
    save_changes
  end
end

Shogun now

  • promoted some attrs to columns
  • automagic attrs wrapper
  • wrap attrs in utility methods

Shogun future


class User < Sequel::Model
  pouch(:preferences) do
    field :favorite_color, type: String
    field :lucky_number, type: Integer
    field :gluten_free?, type: :bool
    field :bff, type: User
  end
end
